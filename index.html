01 · Context

The Netflix Thumbnail Problem

Every time you open Netflix, the platform makes a quiet decision: which image should represent each title on your homepage? That decision is not made by a creative director. It is made by a machine — and it is tested constantly.

Netflix serves personalised artwork to each of its 260+ million members. A thriller might show a chase scene to one viewer and a close-up of the lead actor to another. The goal is always the same: get the right image in front of the right person so they click, start watching, and stay. A difference of even two or three percentage points in click-through rate, multiplied across hundreds of millions of impressions per day, translates directly into viewing hours, subscriber retention, and revenue.

The question this project explores is simple but commercially significant: if you show two different artwork versions to two randomly selected groups of users, can you measure which one actually performs better — and be confident the difference is not just luck?

02 · Method

What Is A/B Testing and Why Does It Matter?

An A/B test is one of the most powerful tools in modern business. The idea is straightforward: take a large group of people, randomly split them in half, show each half a different version of something, measure what happens, and let the data tell you which version wins.

The word randomly is the key. Random assignment is what makes the comparison fair. If you let users self-select which artwork they see, you can never be sure whether a difference in click rates reflects the artwork or some other difference between the kinds of people who chose each version. Randomisation removes that problem by ensuring the two groups are — on average — identical in every way before the experiment begins.

In the context of machine learning and AI, A/B testing is how companies verify that a model improvement actually helps real users. Netflix trains recommendation algorithms and artwork-selection models, but no amount of offline testing tells you for certain how a change will perform in the wild. The experiment is the final check.

Randomly split your audience

Every user is assigned to exactly one group. The assignment is random, so neither group is systematically different from the other before the test begins.

Expose each group to a different version

Group A sees Artwork A. Group B sees Artwork B. Each user only ever sees one version, so there is no contamination between groups.

Measure the outcome

Did the user click? That binary yes/no — one per user — is the raw data the entire analysis is built on.

Test whether the difference is real

A statistical test determines whether the gap in click rates could plausibly have happened by chance, or whether it reflects a genuine performance difference.

03 · Experiment Design

How This Experiment Was Built

This project simulates a Netflix-style artwork experiment in Python using three steps. The simulation mirrors the structure of a real experiment: random assignment, outcome measurement, and statistical evaluation.

Experiment parameters

The experiment assigns 1,000 users equally to two groups. Artwork A has a true underlying click probability of 24% — meaning for every 100 users who see it, approximately 24 will click. Artwork B has a true click probability of 30%. In a real experiment these probabilities are unknown; the test exists to discover them. Here they are set in advance so we can verify that the simulation and statistical machinery work correctly.

How clicks are simulated

Each user's click decision is modelled as a Bernoulli trial — a single weighted coin flip. For a user in Group A, the coin lands on "click" with 24% probability and on "no click" with 76% probability. This is mathematically identical to how a real binary event (clicked or did not click) behaves across a large population.

netflix_ab_test.py 
# ── Step 1: Lock in reproducibility ───────────────────────────────
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import matplotlib.patches as mpatches
from scipy import stats

np.random.seed(7)

# ── Step 2: Define experiment parameters ──────────────────────────
N           = 1000
n_per_group = N // 2
p_artwork_a = 0.24
p_artwork_b = 0.30

# ── Step 3: Create user IDs and assign artwork versions ───────────
user_ids         = np.arange(1, N + 1)
artwork_versions = np.repeat(['Artwork A', 'Artwork B'], n_per_group)

# ── Step 4: Simulate clicks ────────────────────────────────────────
clicks_a = np.random.binomial(n=1, p=p_artwork_a, size=n_per_group)
clicks_b = np.random.binomial(n=1, p=p_artwork_b, size=n_per_group)
clicks   = np.concatenate([clicks_a, clicks_b])

# ── Step 5: Assemble the results DataFrame ────────────────────────
df = pd.DataFrame({
    'user_id':         user_ids,
    'artwork_version': artwork_versions,
    'clicked':         clicks
})

# ── Step 6: Compute group-level metrics ───────────────────────────
summary = (
    df.groupby('artwork_version')['clicked']
    .agg(
        total_users  = 'count',
        total_clicks = 'sum',
        ctr          = 'mean'
    )
    .reset_index()
)

# ── Step 7: Two-proportion z-test ─────────────────────────────────
ctr_a = summary.loc[summary['artwork_version'] == 'Artwork A', 'ctr'].values[0]
ctr_b = summary.loc[summary['artwork_version'] == 'Artwork B', 'ctr'].values[0]

clicks_a_total = summary.loc[summary['artwork_version'] == 'Artwork A', 'total_clicks'].values[0]
clicks_b_total = summary.loc[summary['artwork_version'] == 'Artwork B', 'total_clicks'].values[0]
n_a = n_b = n_per_group

p_pool  = (clicks_a_total + clicks_b_total) / (n_a + n_b)
se      = np.sqrt(p_pool * (1 - p_pool) * (1/n_a + 1/n_b))
z_score = (ctr_b - ctr_a) / se
p_value = 2 * (1 - stats.norm.cdf(abs(z_score)))

diff     = ctr_b - ctr_a
ci_lower = diff - 1.96 * se
ci_upper = diff + 1.96 * se

The chart makes the gap visible. Artwork B outperforms Artwork A by 6 percentage points in absolute terms. But a 6-point gap on its own does not tell us whether the difference is meaningful. Two groups of 500 people will always produce slightly different click rates even if they are seeing exactly the same artwork — that is normal statistical noise. The question is whether the gap is large enough, relative to that noise, to be considered a genuine signal.

05 · Statistical Test

Is the Difference Real or Just Luck?

To answer that question, the code runs a two-proportion z-test — the standard statistical tool for comparing two groups when the outcome is a binary yes/no. Think of it this way: imagine flipping two slightly different coins 500 times each. One lands heads 24% of the time, the other 30%. You'd wonder — is the second coin actually heavier, or did I just happen to get lucky? The z-test answers exactly that question.

The test computes a p-value: the probability that a gap this large — or larger — could have appeared by chance alone, if both artworks actually performed identically. The conventional threshold is 5%. If the p-value falls below 0.05, we say the result is statistically significant, meaning the data is strong enough to trust.

What each number means in plain English

Z-score (~2.24): The observed gap between the two artworks is about 2.24 times larger than the amount of random variation you would normally expect from two groups of this size. Anything above roughly 1.96 clears the significance threshold.

P-value (~0.025): There is approximately a 2.5% chance that this gap happened by luck alone. That is well below the 5% cutoff, so we reject the idea that both artworks perform the same way.

Confidence interval ([+0.8pp, +11.2pp]): We are 95% confident that Artwork B's true advantage over Artwork A is somewhere between 0.8 and 11.2 percentage points. Crucially, the entire interval is above zero — meaning even in the most conservative interpretation of the data, Artwork B still wins.

06 · Business Recommendation

What Netflix Should Do

Primary Recommendation

Ship Artwork B to 100% of users. The experiment produced a statistically significant result with a p-value of approximately 0.025 and a relative lift of 25% over the control. The data provides sufficient confidence to act.

A 6 percentage-point improvement in click-through rate is not a marginal gain at Netflix's scale. Applied across hundreds of millions of daily impressions, even a conservative version of this lift — say, 3 points instead of 6 — would generate tens of millions of additional clicks per day. Those clicks translate into viewing hours, and viewing hours are the primary driver of subscriber retention and renewal.

What to do beyond shipping

Shipping Artwork B is the right immediate decision, but it should not be the last step. Click-through rate is a leading indicator, not a final outcome. The deeper question is whether users who clicked on Artwork B actually enjoyed what they watched. If the artwork attracted clicks by overpromising — showing an action-packed thumbnail for a slow-burn drama, for example — the long-term effect could be negative: users who feel misled watch less, rate worse, and are more likely to cancel.

A complete post-experiment analysis would track play rate (did the click lead to actual viewing?), completion rate (did users finish what they started?), and 30-day retention (are the users who saw Artwork B still subscribing a month later?). Click-through rate opens the door. These downstream metrics tell you whether the experience on the other side was worth walking through it.

Why this methodology scales

The same statistical framework used in this 1,000-user simulation is what Netflix runs across its entire global member base. The inputs change — larger samples, more artwork variants, more complex metrics — but the core logic is identical: randomise, measure, test. That is what makes experimentation one of the most valuable competencies a data-driven organisation can build. The math does not care whether the sample is 1,000 users in a classroom simulation or 10 million users on a Tuesday evening in December. It asks the same question either way: is this real, or is it noise?

This analysis demonstrates that Artwork B produces a genuine, statistically verifiable improvement in user engagement. The recommended action is to deploy Artwork B as the default and continue monitoring downstream retention metrics over the following 30 days.
